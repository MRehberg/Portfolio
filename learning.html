JavaScript Definitions
Term Definition
Conditional
  • A statement that allows the code to make a choice between one or more options and then execute the code associated with that choice to get the desired outcome
  • Generally used when there are only a few choices to select from
Constructor Function
  • A function that defines objects and their features
Event
  • An action or occurrence that happens within the program
  • For example: a user pressing a key, a form being submitted, a mouse click, etc.
  • List of Events
Event Bubbling and Capture
  • Two mechanisms that describe what happens when two event handlers of the same event type are activated on one element
Event Listener
  • A function that attaches an event handler to an element
  • DOM Level 2 Events
Event Object
  • A parameter that is automatically passed to an event handler to provide extra information or features
  • The target property of an event object will be a reference to the element that the event occurred on
Function
  • Stores code that performs a task within an object
JSON
  • JavaScript Object Notation
  • A text-based data format following JavaScript object syntax used for storing and exchanging data
  • Exists as a string and used to transmit data across a network
Object
  • A collection of related properties and methods
Parameters
  • Values included in a function's parentheses to be passed into the function
  • Multiple values should be separated by commas
Prototypes
  • A template object in which methods and properties are defined and made available to other objects
Return Values
  • Values returned by a function after the function has finished executing
Scope
  • Determines the accessibility of variables
  • Local Scope - variables can only be accessed within that function
  • Global Scope - variables can be accessed by all scripts and functions on that page
Switch Statement
  • Evaluates an expression then searches multiple blocks of code to find a match and executes only that block
  • Once execution reaches the break keyword, the switch block ends
  • If no matches are found, a default block will be executed

JavaScript Notes

  • The error "ReferenceError: [ ] is not defined." is a fairly common error. Be sure to check the variable listed in "[ ]" to see if it is in the correct scope. Does the variable need to be local or global?
  • Always be sure to iterate the initializer so it will eventually reach the exit condition, otherwise it will become an infinite loop
  • If you don't want to wait for a loop to finish running to meet the exit condition, you can create a nested if...else conditional within the loop that includes a break statement causing the conditional to end.
  • Don't use event handlers inline with HTML, because it is easier to handle and parse if the JavaScript event handler is located with other JavaScript code. Using an Event Listener will allow you to keep the JavaScript separated from the HTML markup.
